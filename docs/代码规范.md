## 三、Lint 规则（ESLint）

基础配置：`@typescript-eslint/recommended-type-checked` + `eslint-plugin-react` + `eslint-plugin-react-hooks` + `eslint-plugin-tailwindcss` + `eslint-plugin-import`。

关键强制规则：
- `@typescript-eslint/no-explicit-any`: `error`
- `@typescript-eslint/consistent-type-imports`: `error`（`import type { ... }`）
- `import/order`: `error`（`builtin → external → internal → parent → sibling → index`）
- `react-hooks/exhaustive-deps`: `error`
- `tailwindcss/classnames-order`: `warn`
- **自定义规则：禁止在组件中硬编码颜色 / 字号 / 间距，必须走 token**

## 四、命名约定

| 对象 | 规则 | 示例 |
| --- | --- | --- |
| 文件/目录 | `kebab-case` | `event-form.tsx`、`use-theme.ts` |
| React 组件 | `PascalCase` | `EventForm` |
| 组件文件导出 | 默认导出组件同名 | `export default EventForm` |
| Hook | `useXxx` camelCase | `useTheme` |
| 普通函数/变量 | `camelCase` | `getEvents` |
| 常量 | `SCREAMING_SNAKE_CASE` | `MAX_REMINDERS` |
| 类型/接口 | `PascalCase`（**不加 `I` 前缀**） | `type Event`、`interface User` |
| 枚举 | `PascalCase`，成员 `PascalCase` | `TaskStatus.Completed` |
| Zustand store | `useXxxStore` | `useEventStore` |
| TanStack Query key | 数组，第一项 kebab-case | `['events', userId]` |
| CSS 变量 | `--{scope}-{name}` | `--color-bg-primary` |
| 数据库表/列 | `snake_case` | `todo_items`、`created_at` |

## 六、Git 工作流

### 6.1 分支
- `main`：保护分支，始终可发布
- `feat/<scope>` 新功能，如 `feat/event-form`
- `fix/<scope>` 修复
- `refactor/<scope>`
- `chore/<scope>`
- `docs/<scope>`

### 6.2 Commit Message（**Conventional Commits**）

```
<type>(<scope>): <subject>

<body>

<footer>
```

允许的 `type`：`feat` `fix` `refactor` `perf` `docs` `style` `test` `build` `ci` `chore` `revert`
`scope` 示例：`web`、`worker`、`mobile`、`theme`、`cli`。

示例：
```
feat(events): 支持拖拽修改事件时长
fix(web): 主题切换残留颜色
```

由 `commitlint` 在 `commit-msg` 钩子强制。

### 6.3 PR
- 标题遵循 Conventional Commits
- PR 模板勾选 CI 通过、自测截图、无敏感信息
- **Squash merge**，合入后自动删除分支

## 七、主题与样式规则

### 7.1 三层 Token 架构

```
primitive  →  semantic  →  component
（原始色值）   （语义用途）   （组件专属）
```

```ts
// primitive
const primitive = {
  orange50: '#FFF4E6',
  orange500: '#E8956A',
  // ...
};

// semantic
const semantic = {
  colorBgSurface: primitive.orange50,
  colorAccent: primitive.orange500,
  // ...
};

// component（仅当语义不够用才定义）
const component = {
  buttonPrimaryBg: semantic.colorAccent,
};
```

### 7.2 运行时切换

主题以 CSS 变量形式注入 `<html data-theme="...">`，切换主题只替换 `data-theme` 属性。禁止在组件里写 `if (theme === 'dark')`。

### 7.3 组件样式铁律

- **禁止**：`class="bg-[#E8956A]"`、`style={{ color: '#xxx' }}`、Tailwind 任意值硬编码颜色
- **必须**：`class="bg-surface text-primary"`（Tailwind 配置读取 CSS 变量）
- 新组件 = 只消费 semantic token；新功能加主题 = **只改 token 文件**

CI 会用 ESLint 规则 + `stylelint` 扫出违规，直接阻断。

## 八、安全与敏感信息

1. **绝不提交**：`.env`、API Token、OAuth Secret、VAPID Private Key、R2 Access Key
2. **Worker 机密**：`wrangler secret put`，不写入 `wrangler.toml`
3. **GitHub Actions**：用 `secrets.*`，不写入 workflow YAML
4. **Pre-commit**：集成 `gitleaks` 扫描
5. **用户数据**：密码用 `argon2id` 哈希；Session token 存 KV 并设置 TTL
6. **CSP**：Worker 下发严格 CSP 头
7. **速率限制**：Worker 前置基于 IP 的 token bucket

## 九、测试

| 层 | 工具 | 覆盖率门槛 |
| --- | --- | --- |
| 单元测试 | Vitest | ≥ 80%（`packages/shared`） |
| 组件测试 | Vitest + React Testing Library | 关键路径覆盖 |
| 端到端 | Playwright | 登录 / 创建 TODO / 创建 Event / 主题切换 |
| 契约测试 | tRPC 类型推导本身即契约 | — |

**规则**：
- 每个 bug 修复必须附带回归测试
- `test` 文件与被测文件同目录，后缀 `.test.ts(x)`
- 禁止 `test.only` / `describe.only` 进主干

## 十、文档注释

- **公共 API**（`packages/*` 的导出、tRPC procedures）**必须**写 TSDoc
- 复杂算法/非显然逻辑必须写 `// why:` 注释解释**为什么**，而非**是什么**
- README 与 `docs/` 保持与代码同步

---

## 十一、依赖管理

- 新增依赖必须在 PR 中说明理由
- 优先挑选：TypeScript 原生、Tree-shakeable、License 兼容（MIT / Apache-2.0 / BSD）
- 禁止 GPL 类传染性许可（除非隔离进独立工具）
- `pnpm audit` 出现 `high` 及以上漏洞阻断 CI
- 每季度用 `pnpm outdated` 评估升级

## 十二、提交前检查清单

```bash
pnpm lint        # ESLint
pnpm typecheck   # tsc --noEmit
pnpm test        # vitest run
pnpm format      # prettier --check
```

以上四条 CI 同样会跑。**本地通过 ≠ CI 一定通过**（OS 差异），以 CI 结果为准。
