# 项目介绍

> 应用名：**Kairo**（/ˈkaɪroʊ/，取自希腊语 *Kairos* —— "恰当的时机"）
> npm / 包名：`kairo`
> 文档版本：v1.1 · 2026-04-23

---

## 一、项目定位

**Kairo** 是一个**跨平台、多用户、Serverless** 的日程与任务管理系统。在原 Python + Tkinter 单机版的基础上重构，核心理念：

- **把复杂任务拆分成可执行的小步骤** —— 继承原版"任务分离、逐个击破"的哲学
- **TODO 与 Event 双轨并行** —— 可顺延的待办 + 固定时间的事件，两者可互相绑定
- **多端无缝同步** —— Web / Windows / macOS / Linux / Android 数据实时一致
- **Serverless 零运维** —— 全部部署在 Cloudflare，管理员通过 CLI 一键完成配置
- **主题可插拔** —— 新增功能仅需编辑主题 token 文件即可完成视觉适配

## 二、目标用户

| 角色 | 权限 |
| --- | --- |
| 管理员（Admin） | 配置登录方式、地图 API、IP/地区访问限制、用户管理、主题默认值 |
| 普通用户（User） | 创建/管理自己的 TODO、Event，选择主题，绑定多设备 |
| 访客 | 仅能看到登录/注册入口（取决于管理员配置） |

## 三、核心功能

### 3.1 账号与登录
- **多用户**：注册、登录、个人设置
- **登录方式可由管理员按需启用**：
  - 邮箱 + 密码
  - GitHub OAuth
  - Google OAuth
  - Cloudflare Access（Zero Trust）
  - 设备配对码（用于移动端快速登录）
- **访问控制**：管理员可配置 IP/地区白名单或黑名单（基于 CF Worker `cf.country` 与 `cf.connectingIp`）

### 3.2 任务模型

```
TODO（待办）                       Event（事件）
├─ 可顺延到第二天                  ├─ 固定日期/时间
├─ 可嵌套子 TODO                   ├─ 可设置位置、提醒、重复规则（RRULE）
├─ 完成所有子 TODO 才能完成父 TODO   └─ 创建时可自动生成绑定的 TODO
└─ 可绑定一个 Event
```

字段（TODO 与 Event 共有 + 各自私有）：

| 字段 | 类型 | 归属 | 说明 |
| --- | --- | --- | --- |
| `id` | ULID | 共有 | 主键 |
| `title` | string | 共有 | 标题 |
| `description` | markdown | 共有 | 详情（支持 Markdown） |
| `createdAt` / `updatedAt` | ISO8601 | 共有 | 审计字段 |
| `version` | int | 共有 | 乐观锁版本号 |
| `parentId` | ULID? | TODO | 父 TODO |
| `completed` | bool | TODO | 完成状态 |
| `rollover` | bool | TODO | 是否自动顺延 |
| `linkedEventId` | ULID? | TODO | 绑定的 Event |
| `startAt` / `endAt` | ISO8601 | Event | 起止时间 |
| `location` | `{ text, lat?, lng? }` | Event | 位置（文本或地图选点） |
| `reminders` | `Reminder[]` | Event | 提前多久提醒（多条） |
| `rrule` | string | Event | 重复规则（iCal RRULE） |
| `timezone` | IANA tz | Event | 默认 `Asia/Shanghai` (UTC+8) |

### 3.3 提醒

三端同时支持：
- **Web** —— Web Push（VAPID）
- **桌面端** —— 原生系统通知（Tauri notification）
- **移动端** —— Android 本地通知 + FCM 备份
- **邮件** —— Cloudflare Email Workers 发送

### 3.4 周视图（空闲时间表）

- **周一 ~ 周日** 为一周
- 网格展示：纵轴时间（0:00 - 24:00），横轴 7 天
- **拖拽**：
  - 拖拽事件块 → 改日期/时间
  - 拖拽边缘 → 改时长
  - 横向拖拽整个视图 → 切换周
- 空白区域即为**空闲时段**，一目了然

### 3.5 Android 桌面小组件

- **显示当月迷你日历 + 今日事件列表**
- 点击事件 → 弹出**透明悬浮窗**（Android Overlay），显示详情 / 编辑 / 删除
- 通过 RN 原生模块 + Kotlin 实现 `AppWidgetProvider`

### 3.6 位置功能

- 默认使用 **OpenStreetMap Nominatim**（免费、开源）地理编码
- 使用 **MapLibre GL** 渲染地图
- 管理员面板可切换为高德地图或 Google Maps（需自备 API Key）
- 用户可直接输入纯文本位置，或在地图上选点

### 3.7 主题系统

- 预置 **4 套主题**（详见 [主题设计稿](./主题设计.md)，将分批产出）
  1. **暖色日系**（继承原版，亚麻暖白 + 珊瑚橙）
  2. **晨雾莫奈**（淡蓝 + 藕粉，水彩风）
  3. **深色极夜**（OLED 友好纯黑 + 电光青）
  4. **纸质书感**（米白 + 黑 + 朱砂红点缀）
- 所有主题遵循**三层 Token 架构**（primitive → semantic → component）
- 支持**跟随系统明暗**
- `[可选]` 自定义主题编辑器（用户自调色）

### 4.1 Monorepo 结构

```
scheduler/
├── apps/
│   ├── web/         # Vite + React PWA
│   ├── desktop/     # Tauri
│   ├── mobile/      # Expo (React Native)
│   │   └── android/ # 原生 Widget 模块 (Kotlin)
│   └── worker/      # Cloudflare Worker API
├── packages/
│   ├── shared/      # 类型、schema、业务纯函数
│   ├── ui/          # 跨端共用的设计系统组件
│   └── themes/      # 主题 token 文件
├── tools/
│   └── cli/         # 部署/更新 CLI
├── docs/
├── .github/
│   └── workflows/   # CI/CD
└── pnpm-workspace.yaml
```

### 4.2 数据存储选型

| 存储 | 用途 |
| --- | --- |
| **Cloudflare D1** | 主数据库（用户、TODO、Event、提醒） |
| **R2** | 任务附件、用户头像、JSON 全量备份/导出 |
| **KV** | Session、管理员配置、限流计数 |
| **Queues** | 提醒调度、顺延任务 Cron |
| **Durable Objects** | 可选 · 用于跨设备实时推送（v2） |

### 4.3 一致性策略

- 服务端每条记录带 `version` 字段，写入时校验（乐观锁）
- 客户端提交过时版本 → 返回 `409 Conflict` + 服务端最新版
- 客户端展示冲突解决 UI（保留我的 / 保留远程 / 合并）
- 离线编辑 → 进队列 → 恢复网络后批量同步

## 五、部署与发布

### 5.1 管理员一键部署

```bash
npx @scheduler/cli init
```

CLI 将引导：
1. 登录 Cloudflare（OAuth 或 API Token）
2. 选择域名 → 输入/选择子域名
3. 自动创建 D1、R2 bucket、KV namespace、Queue
4. 部署 Worker、上传 Web 静态资源
5. 生成一次性 setup URL（含 token）
6. 访问 URL 完成管理员密码、登录方式、地图 API、访问限制的向导配置

### 5.2 客户端分发

| 平台 | 方式 |
| --- | --- |
| Web | PWA，用户直接访问域名 |
| Windows/macOS/Linux | 域名首页提供 Tauri 安装包下载 |
| Android | 域名首页提供 APK 直链 + 二维码（**不上架 Play Store**） |

### 5.3 更新流程

```bash
npx @scheduler/cli deploy
```

- 推送最新代码到 GitHub
- GitHub Actions 自动构建 Web / APK / 桌面安装包
- Worker 通过 `wrangler deploy` 更新
- 敏感信息（API Token、VAPID key、OAuth secret）只存在 `wrangler secret` 与 GitHub Actions Secrets，**绝不进仓库**
